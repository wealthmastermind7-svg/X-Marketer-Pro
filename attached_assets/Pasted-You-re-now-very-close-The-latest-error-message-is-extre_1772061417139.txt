# 🎯 What Is Actually Happening

You currently have:

```ts
app.get("/", (req, res) => {
  res.status(200).send("ok");
});

configureExpoAndLanding(app);
```

So:

1. Health check hits `/`
2. Express matches the first `/`
3. Returns `"ok"`
4. Expo landing middleware never runs
5. Replit detects routing inconsistency or misconfigured middleware
6. Promotion fails

Replit now expects the full routing stack to be initialized before responding.

---

# 🔥 The Correct Architecture

You must:

1. Initialize all middleware first
2. Then define the health check route
3. Ensure health check does not interfere with landing routes

---

# ✅ Proper Order

Your `server/index.ts` should look like this:

```ts
// 1️⃣ Create app
const app = express();

// 2️⃣ Register ALL middleware first
app.use(express.json());
app.use(express.urlencoded({ extended: true }));

await configureExpoAndLanding(app);

// 3️⃣ Then define health endpoint
app.get("/health", (req, res) => {
  res.status(200).send("ok");
});

// 4️⃣ Then define fallback root handler LAST
app.get("/", (req, res) => {
  res.status(200).send("ok");
});

// 5️⃣ Start server
app.listen(process.env.PORT);
```
